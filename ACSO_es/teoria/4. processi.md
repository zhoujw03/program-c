# 进程（Process）核心理论+实战示例
## 一、进程的定义
进程是**运行中的程序**在操作系统中的动态实体，是操作系统**资源分配和CPU调度的基本单位**。
- 程序：存储在硬盘上的静态二进制文件/代码，无运行状态，不占用系统资源；
- 进程：程序被加载到内存后，包含**代码段、数据段、内存空间、文件描述符、PID/PPID**等，拥有独立的运行环境，由操作系统调度执行。
### 进程核心标识
- **PID**：进程唯一ID，操作系统通过PID区分不同进程；
- **PPID**：父进程ID，标记当前进程由哪个进程创建；
- **进程状态**：描述进程的运行状态（R/S/D/t/Z）；
- **资源占用**：CPU、内存（VIRT虚拟内存/RSS物理内存）占用情况。

## 二、进程的查看方式（终端命令）
无图形化工具依赖，纯终端命令即可查看进程全量信息，重点关注**PID/PPID/状态/执行命令**，常用命令如下：
| 命令          | 核心功能                     | 查看重点                     |
|---------------|------------------------------|------------------------------|
| `ps aux`      | 查看系统所有进程完整信息     | PID、PPID、%CPU、%MEM、状态、执行命令 |
| `ps -ef`      | 按格式显示进程，突出父子关系 | 直接列显PID、PPID，快速看进程层级 |
| `pstree`      | 以树状结构显示进程关系       | 直观看到整个系统的进程树，根为PID=1 |
| `pgrep 进程名`| 按程序名快速查询进程PID      | 如`pgrep main`，直接输出目标进程PID |
| `ps aux \| grep 进程名` | 过滤查看指定进程 | 精准定位目标进程，排除无关进程干扰 |

### 示例输出（ps -f 查看父子进程）
```bash
UID        PID  PPID  C STIME TTY          TIME CMD
root         1     0  0  2024 ?        00:00:02 /sbin/init
ubuntu    1234     1  0  2024 ?        00:00:05 /usr/bin/sshd
ubuntu    5678  1234  0  2024 pts/0    00:00:00 bash
ubuntu    8910  5678  0 10:00 pts/0    00:00:01 ./main  # 自定义程序，父进程为bash（5678）
```

## 三、进程的父子关系与层级制度
### 1. 基本层级结构
操作系统中所有进程形成**以根进程为起点的树状结构**，核心规则：
- 根进程：系统启动后第一个进程，**PID=1**（Linux为`init`/`systemd`，macOS为`launchd`），所有进程最终都是其子孙进程；
- 父子进程：进程通过`fork()`创建新进程，创建者为**父进程**，被创建者为**子进程**，子进程的PPID等于父进程的PID；
- 层级继承：子进程会**默认继承**父进程的环境变量、文件描述符、工作目录、用户权限、信号处理方式等资源。

### 2. 两种特殊进程（重要）
#### （1）孤儿进程 (processi orfano)
- 定义：**父进程先于子进程退出**，子进程失去父进程后，会被操作系统的根进程（PID=1）收养；
- 核心特征：子进程的**PPID会被修改为1**，无其他特殊状态；
- 特性：无害，根进程会负责回收其资源，不会占用系统进程表。

#### （2）僵尸进程 (processi zombie)
- 定义：**子进程先于父进程退出**，但父进程未调用`wait()`/`waitpid()`回收子进程的资源，子进程残留进程表项，状态变为**Z（Zombie）**；
- 核心特征：PID存在、状态为Z、无实际运行代码，仅占用进程表资源；
- 危害：系统进程表容量有限，大量僵尸进程会导致无法创建新进程；
- 解决方式：① 父进程主动调用`wait()`/`waitpid()`回收；② 杀死父进程，子进程被PID=1收养后自动回收。

### 示例（僵尸进程创建代码）
```c
#include<stdio.h>
#include<stdlib.h>
#include<unistd.h>
#include<sys/types.h>
#include<sys/wait.h>
int main(){
    pid_t pid = fork();
    if( pid == 0 ){
        // 子进程：睡眠30秒后退出，成为僵尸进程
        printf("子进程：PID=%d，PPID=%d，即将退出\n", getpid(), getppid());
        sleep(30);
        exit(22); // 子进程退出，父进程未wait
    }
    else{
        // 父进程：睡眠10秒后退出，未回收子进程
        printf("父进程：PID=%d，子进程PID=%d，不执行wait\n", getpid(), pid);
        sleep(10);
        exit(0);
    }
}
```
编译运行后，用`ps aux | grep 程序名`可看到子进程状态为**Z**。

## 四、进程的核心操作流程（创建/等待/结束）
所有操作均基于Linux/macOS系统调用，C语言直接调用即可，以下为**完整流程+可运行代码示例**，重点掌握函数原型、返回值、核心逻辑。

### 1. 进程创建：`fork()`
#### 函数原型
```c
#include <unistd.h>
pid_t fork(void);
```
#### 核心作用
复制当前进程（父进程），创建一个全新的子进程，父子进程共享代码段，数据段采用**写时复制（COW）**（只有当一方修改数据时，才会为其分配独立内存）。
#### 返回值规则（**必须牢记**）
- 父进程：返回**子进程的PID**（大于0的整数）；
- 子进程：返回**0**；
- 失败：返回**-1**（如系统进程数达到上限）。
#### 执行特性
`fork()`后，父子进程的执行顺序由**操作系统CPU调度**决定，无固定先后，需通过代码控制（如`sleep()`）。

#### 完整示例（fork创建父子进程）
```c
#include<stdio.h>
#include<stdlib.h>
#include<unistd.h>
#include<sys/types.h>
int main(){
    pid_t pid = fork(); // 创建子进程
    if(pid == -1){ // fork失败处理
        perror("fork创建进程失败"); //perror 用来打印错误输出
        exit(1);
    }
    else if(pid == 0){ // 子进程逻辑（返回值为0）
        printf("CHILD: PID: %5d，PPID: %5d\n", getpid(), getppid());
        sleep(2); // 模拟子进程工作
        printf("CHILD: 执行完成，即将退出\n");
        exit(0); // 子进程正常退出
    }
    else{ // 父进程逻辑（返回值为子进程PID）
        printf("PARENT: PID: %5d，创建的子进程PID: %5d\n", getpid(), pid);
        sleep(3); // 确保父进程在子进程后执行
        printf("PARENT: 执行完成\n");
    }
    return 0;
}
```
#### 编译运行
```bash
gcc fork_demo.c -o fork_demo
./fork_demo
```
#### 输出结果
```
PARENT: PID:  1234，创建的子进程PID:  1235
CHILD: PID:  1235，PPID:  1234
CHILD: 执行完成，即将退出
PARENT: 执行完成
```

### 2. 进程等待：`wait()` / `waitpid()`
#### 核心作用
父进程调用该函数，**阻塞等待子进程退出**，并回收子进程的资源，**避免子进程变成僵尸进程**；同时可通过函数参数获取子进程的退出状态（退出码/终止信号）。
#### 函数原型
```c
#include <sys/wait.h>
// 等待任意一个子进程退出，阻塞式（无子进程退出则一直等待）
pid_t wait(int *wstatus);
// 等待指定PID的子进程，可设置阻塞/非阻塞，灵活性更高
pid_t waitpid(pid_t pid, int *wstatus, int options);
```
#### 关键参数说明
- `pid`：指定等待的子进程PID，`-1`表示等待**任意一个**子进程（与`wait()`功能一致）；
- `wstatus`：指向整型的指针，用于**保存子进程的退出状态**，可通过系统宏解析；
- `options`：阻塞模式，`0`为**阻塞式**，`WNOHANG`为**非阻塞式**（无子进程退出则立即返回0）。

#### 退出状态解析宏（**必须掌握**）
通过`wstatus`解析子进程退出原因，返回值为**非0表示真**，0表示假：
| 宏                | 核心作用                     |
|-------------------|------------------------------|
| `WIFEXITED(wstatus)` | 判断子进程是否**正常退出**（如`exit()`） |
| `WEXITSTATUS(wstatus)` | 获取子进程正常退出的**退出码**（如`exit(22)`则返回22） |
| `WIFSIGNALED(wstatus)` | 判断子进程是否**被信号终止**（如`kill -9`） |
| `WTERMSIG(wstatus)` | 获取终止子进程的**信号编号**（如SIGKILL为9） |

#### 完整示例（waitpid等待指定子进程）
```c
#include<stdio.h>
#include<stdlib.h>
#include<unistd.h>
#include<sys/types.h>
#include<sys/wait.h>
int main(){
    pid_t pid = fork();
    int status; // 保存子进程退出状态
    if(pid == -1){
        perror("fork失败");
        exit(1);
    }
    else if(pid == 0){
        // 子进程：模拟运行后退出，退出码33
        printf("子进程PID: %d，PPID: %d，退出码33\n", getpid(), getppid());
        sleep(2);
        exit(33);
    }
    else{
        // 父进程：等待指定子进程（pid），阻塞式
        printf("父进程PID: %d，等待子进程%d退出\n", getpid(), pid);
        waitpid(pid, &status, 0); // 阻塞等待
        
        // 解析退出状态
        if(WIFEXITED(status)){
            printf("子进程正常退出，退出码：%d\n", WEXITSTATUS(status));
        }
        if(WIFSIGNALED(status)){
            printf("子进程被信号终止，信号号：%d\n", WTERMSIG(status));
        }
    }
    return 0;
}
```
#### 输出结果
```
父进程PID: 1236，等待子进程1237退出
子进程PID: 1237，PPID: 1236，退出码33
子进程正常退出，退出码：33
```

### 3. 进程结束：`exit()` / `_exit()`
#### 核心作用
主动终止当前进程，释放进程占用的资源（内存、文件描述符等），并向父进程传递**退出码**（0表示正常退出，非0表示异常退出）。
#### 函数原型
```c
#include <stdlib.h> // exit() 头文件
#include <unistd.h> // _exit() 头文件
// 库函数，清理用户态缓冲区后退出（推荐使用）
void exit(int status);
// 系统调用，直接退出，不清理缓冲区（底层调用）
void _exit(int status);
```
#### 核心区别
- `exit()`：退出前会**刷新并关闭标准IO缓冲区**（如`printf`未换行的内容会被输出），再调用`_exit()`；
- `_exit()`：直接终止进程，**不处理缓冲区**，适合内核态/特殊场景。
#### 示例（exit与缓冲区差异）
```c
#include<stdio.h>
#include<stdlib.h>
#include<unistd.h>
int main(){
    printf("使用exit退出，会刷新缓冲区"); // 无换行，缓冲区未刷新
    exit(0); // 退出前刷新缓冲区，内容会输出
    // _exit(0); // 直接退出，缓冲区不刷新，内容不会输出
    return 0;
}
```

## 五、exec系列函数：替换进程代码（不改变PID）
### 1. 核心作用
在**当前进程的PID不变**的前提下，用**新程序的代码段、数据段、内存空间**完全替换当前进程的内容，实现“在当前进程中执行另一个程序”。
### 2. 关键特性
- **PID不变**：仅替换进程的运行实体，进程的PID、PPID等标识保持不变；
- **原代码失效**：若`exec`执行成功，当前进程中`exec`之后的代码**不会被执行**（因为代码段已被替换）；
- **常用搭配**：与`fork()`配合使用——父进程`fork()`创建子进程，子进程通过`exec`替换为新程序，父进程用`wait()`回收子进程，这是Linux/macOS中创建新进程执行外部程序的**经典流程**。

### 3. 常用exec函数（按需选择）
均位于`<unistd.h>`头文件，核心差异在于**参数传递方式**和**程序路径查找方式**，最常用`execl`和`execvp`：
| 函数        | 参数特点               | 路径查找方式       |
|-------------|------------------------|--------------------|
| `execl`     | 列表传参，NULL结尾     | 需指定**绝对/相对路径** |
| `execlp`    | 列表传参，NULL结尾     | 自动从PATH环境变量查找 |
| `execv`     | 数组传参，数组必须以NULL结尾     | 需指定**绝对/相对路径** |
| `execvp`    | 数组传参               | 自动从PATH环境变量查找 |

#### 最常用函数：`execl`
```c
int execl(const char *path, const char *arg0, const char *arg1, ..., NULL);
```
- `path`：要执行的新程序的**绝对/相对路径**（如`./pluto`、`/bin/ls`）；
- `arg0~argn`：传递给新程序的参数，**arg0必须为程序名本身**，最后一个参数必须为`NULL`（标记参数结束）；
- 返回值：若执行成功，无返回值；若执行失败，返回**-1**（如程序不存在、路径错误），并执行后续代码。

### 4. 完整示例（fork+exec经典搭配）
分为两个程序：**pippo.c**（父进程，创建子进程并执行exec）、**pluto.c**（被exec替换的新程序），实现子进程替换为新程序并传递参数。

#### 程序1：pluto.c（被执行的新程序，接收并打印参数）
```c
#include<stdio.h>
#include<stdlib.h>
#include<unistd.h>
int main(int argc,char **argv){
    // argc：参数个数；argv：参数数组
    printf("【pluto】参数个数argc: %d\n", argc);
    for (int i = 0; i < argc; i++ ){
        printf("【pluto】argv[%d]: %s\n", i, argv[i]);
    }
    exit(45); // 新程序退出，退出码45，供父进程获取
}
```

#### 程序2：pippo.c（父进程，fork+exec调用pluto）
```c
#include<stdio.h>
#include<stdlib.h>
#include<unistd.h>
#include<sys/types.h>
#include<sys/wait.h>
int main(){
    pid_t pid = fork();
    int status;
    if(pid == 0){
        // 子进程：执行exec，替换为./pluto，传递参数"pluto"、"10"、"20"
        printf("【子进程/pippo】PID: %d，执行exec替换为pluto\n", getpid());
        execl("./pluto", "pluto", "10", "20", NULL);
        
        // 只有exec执行失败时，才会执行以下代码
        printf("【子进程/pippo】exec执行失败（如pluto不存在）\n");
        abort(); // 异常终止进程
    }
    else{
        // 父进程：等待子进程执行完成，回收资源并获取退出码
        printf("【父进程/pippo】PID: %d，子进程PID: %d\n", getpid(), pid);
        wait(&status);
        if(WIFEXITED(status)){
            printf("【父进程/pippo】pluto退出码：%d\n", WEXITSTATUS(status));
        }
        exit(0);
    }
}
```

## 六、信号（Signal）：进程的通信与控制（含信号篡改/自定义）
### 1. 信号的定义
信号是操作系统向进程发送的**异步通知**，用于控制进程的行为（如终止、暂停、继续），是进程间最简洁的通信方式。
- 无数据传递：信号仅传递“事件通知”，不携带任何数据；
- 异步执行：进程接收到信号后，会中断当前执行流程，先处理信号，再恢复原流程；
- 处理方式：进程对信号有3种处理方式——**默认处理**（系统预设，如终止进程）、**忽略信号**（SIG_IGN）、**自定义处理**（篡改信号，用自定义函数替代默认行为）。

### 2. 常用信号
信号有固定的**信号名**和**数字编号**，以下为开发中最常用的信号，Linux/macOS通用：
| 信号名   | 数字 | 核心作用                     | 触发方式                  | 能否自定义/忽略 |
|----------|------|------------------------------|---------------------------|----------------|
| SIGINT   | 2    | 中断进程                     | 终端按`Ctrl+C`            | 能             |
| SIGTERM  | 15   | 正常终止进程（默认）         | `kill 进程PID`（无参数）  | 能             |
| SIGKILL  | 9    | 强制终止进程                 | `kill -9 进程PID`         | 不能           |
| SIGCHLD  | 17   | 子进程状态变化（退出/暂停）  | 子进程`exit()`/被信号终止 | 能             |
| SIGSTOP  | 19   | 暂停进程                     | `kill -19 进程PID`        | 不能           |
| SIGCONT  | 18   | 继续运行被暂停的进程         | `kill -18 进程PID`        | 能             |
| SIGUSR1  | 10   | 用户自定义信号1              | `kill -USR1 进程PID`      | 能             |
| SIGUSR2  | 12   | 用户自定义信号2              | `kill -USR2 进程PID`      | 能             |
| SIGALRM  | 14   | 定时信号                     | 程序调用`alarm()`         | 能             |

### 3. 信号的核心操作函数
#### （1）信号注册/篡改：`signal()`
##### 函数原型
```c
#include <signal.h>
// 定义信号处理函数指针类型
typedef void (*sighandler_t)(int);
// 为指定信号设置处理方式（核心：篡改信号的默认行为）
sighandler_t signal(int sig, sighandler_t handler);
```
##### 关键参数
- `sig`：要处理的信号（如SIGINT、SIGUSR1）；
- `handler`：信号的处理方式，有3种取值：
  1. **自定义函数名**：篡改信号，用自定义函数处理（如`sigusr1_handler`）；
  2. **SIG_IGN**：忽略该信号（进程接收到后不做任何处理）；
  3. **SIG_DFL**：恢复该信号的**系统默认处理方式**（取消自定义/忽略）。
##### 返回值
- 成功：返回该信号**之前的处理方式**（如之前的自定义函数指针）；
- 失败：返回**SIG_ERR**（宏定义，值为-1）。

#### （2）信号发送：`kill()`
##### 函数原型
```c
#include <signal.h>
#include <sys/types.h>
// 向指定PID的进程发送指定信号
int kill(pid_t pid, int sig);
```
##### 作用
程序中主动向其他进程发送信号，终端命令`kill`本质就是调用该函数。
##### 返回值
- 成功：返回0；
- 失败：返回-1（如进程不存在、无权限发送信号）。

#### （3）定时信号：`alarm()`
##### 函数原型
```c
#include <unistd.h>
// 设定定时，n秒后向当前进程发送SIGALRM信号
unsigned int alarm(unsigned int seconds);
```
##### 特性
- `alarm(5)`：5秒后向当前进程发送SIGALRM信号，默认处理方式为**终止进程**；
- `alarm(0)`：取消当前所有未触发的定时信号；
- 定时为**一次性**，触发后自动失效。

### 4. 信号实战示例（核心场景：自定义/忽略/恢复/定时）
以下为4个高频信号使用场景，均为**可运行完整代码**，覆盖**信号篡改、信号忽略、信号恢复、定时信号**，对应开发中最常用的信号操作。

#### 示例1：自定义处理SIGINT（篡改Ctrl+C，拒绝进程退出）
默认情况下，`Ctrl+C`发送SIGINT信号，进程会终止；通过`signal()`自定义处理函数，篡改该信号，实现“按Ctrl+C不退出，仅打印提示”。
```c
#include<stdio.h>
#include<stdlib.h>
#include<signal.h>
#include<unistd.h>

// 自定义SIGINT处理函数（篡改信号）
void sigint_handler(int sig){
    // sig为接收到的信号编号（SIGINT为2）
    printf("接收到SIGINT信号（%d），拒绝退出！\n", sig);
}

int main(){
    // 为SIGINT设置自定义处理函数（篡改默认行为）
    if(signal(SIGINT, sigint_handler) == SIG_ERR){
        perror("signal注册失败");
        exit(1);
    }
    printf("进程PID: %d，按Ctrl+C测试自定义信号（无限循环）\n", getpid());
    // 无限循环，保持进程运行
    while(1){
        sleep(1);
    }
    return 0;
}
```
**运行效果**：按`Ctrl+C`，进程不退出，仅打印自定义提示。

#### 示例2：忽略SIGUSR2 + 自定义SIGUSR1 + 恢复默认（信号的完整篡改流程）
实现：① 自定义SIGUSR1，接收4次后恢复默认；② 忽略SIGUSR2，接收到后不做任何处理；③ 终端用`kill -USR1/-USR2 进程PID`发送信号测试。
```c
#include<stdio.h>
#include<stdlib.h>
#include<signal.h>
#include<unistd.h>
typedef void (*sighandler_t)(int);

int go = 0; // 记录SIGUSR1接收次数

// 自定义SIGUSR1处理函数
void sigusr1_handler(int sig){
    go++;
    printf("接收到SIGUSR1，计数：%d\n", go);
    // 接收4次后，恢复SIGUSR1的默认处理方式（篡改后恢复）
    if(go == 4){
        signal(SIGUSR1, SIG_DFL);
        printf("SIGUSR1已恢复默认处理（再次发送会终止进程）\n");
    }
}

int main(){
    printf("进程PID: %d\n", getpid());
    // 1. 自定义SIGUSR1（篡改）
    signal(SIGUSR1, sigusr1_handler);
    // 2. 忽略SIGUSR2（SIG_IGN）
    signal(SIGUSR2, SIG_IGN);
    printf("已自定义SIGUSR1（4次后恢复）、忽略SIGUSR2\n");
    printf("终端执行：kill -USR1 %d / kill -USR2 %d\n", getpid(), getpid());
    
    // 无限循环，保持进程运行
    while(1){
        sleep(1);
    }
    return 0;
}
```
**运行效果**：
- 前4次发送`kill -USR1 PID`：执行自定义函数，打印计数；
- 第5次发送`kill -USR1 PID`：恢复默认，进程终止；
- 任意次数发送`kill -USR2 PID`：进程无任何反应（忽略）。

#### 示例3：捕获SIGCHLD信号，自动回收子进程（解决僵尸进程）
子进程退出时会向父进程发送SIGCHLD信号，通过自定义该信号的处理函数，在函数中调用`waitpid()`**非阻塞回收所有退出的子进程**，实现“子进程退出即回收，永不产生僵尸进程”。
```c
#include<stdio.h>
#include<stdlib.h>
#include<signal.h>
#include<unistd.h>
#include<sys/wait.h>

// 自定义SIGCHLD处理函数：非阻塞回收所有子进程
void sigchld_handler(int sig){
    // WNOHANG：非阻塞，无子进程退出则立即返回0
    while(waitpid(-1, NULL, WNOHANG) > 0);
    printf("子进程已退出，已自动回收（无僵尸进程）\n");
}

int main(){
    // 注册SIGCHLD信号，自定义处理（自动回收子进程）
    signal(SIGCHLD, sigchld_handler);
    pid_t pid = fork();
    if(pid == 0){
        // 子进程：睡眠2秒后退出，触发SIGCHLD
        printf("子进程PID: %d，2秒后退出\n", getpid());
        sleep(2);
        exit(0);
    }
    else{
        printf("父进程PID: %d，不主动调用wait（靠信号自动回收）\n", getpid());
        // 无限循环，保持父进程运行
        while(1){
            sleep(1);
        }
    }
    return 0;
}
```
**运行效果**：子进程退出后，父进程通过SIGCHLD信号自动回收，用`ps aux`查看无僵尸进程。

#### 示例4：定时信号SIGALRM，实现程序超时控制
通过`alarm(5)`设置5秒定时，5秒后向当前进程发送SIGALRM信号；自定义SIGALRM处理函数，实现“程序运行超时则执行自定义逻辑（如退出）”。
```c
#include<stdio.h>
#include<stdlib.h>
#include<signal.h>
#include<unistd.h>

// 自定义SIGALRM处理函数：程序超时提示并退出
void sigalrm_handler(int sig){
    printf("程序运行超时（5秒），接收到SIGALRM信号，强制退出！\n");
    exit(1); // 超时后异常退出
}

// 模拟程序运行的函数
void run_task(int seconds){
    for(int i=seconds; i>0; i--){
        printf("程序运行中，剩余%d秒\n", i);
        sleep(1);
    }
    printf("程序运行完成，无超时！\n");
}

int main(){
    // 自定义SIGALRM信号（篡改默认终止行为）
    signal(SIGALRM, sigalrm_handler);
    // 设置5秒定时，5秒后发送SIGALRM
    alarm(5);
    // 模拟程序运行：传入参数为运行秒数（如传入10则超时，传入3则正常）
    run_task(10);
    // 程序正常完成，取消定时信号
    alarm(0);
    return 0;
}
```
**运行效果**：
- 传入`run_task(10)`：5秒后触发SIGALRM，执行自定义函数，程序退出；
- 传入`run_task(3)`：3秒后程序运行完成，调用`alarm(0)`取消定时，无信号触发。

## 七、进程的完整生命周期（总结）
结合以上所有操作，一个进程的完整生命周期为**经典fork-exec-wait流程**，也是Linux/macOS中进程创建和运行的标准流程：
1. 父进程调用`fork()`创建子进程，子进程继承父进程的资源；
2. 子进程调用`exec()`系列函数，将自身代码替换为新程序（PID不变）；
3. 新程序执行业务逻辑，执行完成后通过`exit()`主动退出，或被信号终止；
4. 子进程退出时，向父进程发送SIGCHLD信号；
5. 父进程通过`wait()`/`waitpid()`（或捕获SIGCHLD信号）回收子进程资源，避免僵尸进程；
6. 父进程执行业务逻辑后，通过`exit()`退出；
7. 若父进程先于子进程退出，子进程被根进程（PID=1）收养，成为孤儿进程，由根进程负责回收。

## 八、核心知识点速记（复习重点）
1. `fork()`创建子进程，返回值区分父子（父返子PID，子返0），数据写时复制；
2. `wait()`/`waitpid()`回收子进程，解析退出状态用WIFEXITED/WTERMSIG等宏；
3. `exec()`替换进程代码，PID不变，执行成功后后续代码失效，常与`fork()`搭配；
4. 孤儿进程PPID=1，无害；僵尸进程状态Z，需父进程wait回收；
5. 信号是异步通知，`signal()`可自定义/忽略/恢复信号，SIGKILL/SIGSTOP不能被修改；
6. 子进程退出发送SIGCHLD信号，捕获该信号可实现子进程自动回收；
7. `alarm()`设置定时信号SIGALRM，用于实现程序超时控制。
